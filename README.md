# Splendor RL

基于 **Stable-Baselines3 MaskablePPO** 的 2 人无贵族 **璀璨宝石 (Splendor)** 强化学习训练项目。采用**自我对弈 (Self-Play)** 对手池机制，结合**势能奖励塑形 (Potential-Based Reward Shaping, PBRS)**，迭代训练能在真实对局中自己变强的 AI 代理。

---

## 快速开始

### 训练环境要求

- Python ≥ 3.10
- CUDA 支持的 GPU（推荐 RTX 3060+ / 16GB VRAM，纯 CPU 训练也可运行但较慢）
- 32GB+ RAM（Windows spawn 模式下 22 个并行环境会占用约 6-8GB）

### 安装

```bash
# 1. 创建虚拟环境
python -m venv .venv

# 2. 激活虚拟环境 (Windows)
.venv\Scripts\activate

# 3. 安装依赖
pip install -r requirements.txt
```

### 运行训练

```bash
# 完整自我对弈训练（100 代、约 50M 步）
python scripts/train.py

# 快速测试（5 代、缩减配置，约 2-3 分钟）
python scripts/train.py --test

# 仅随机对手基线
python scripts/train.py --baseline

# 从检查点恢复
python scripts/train.py --resume checkpoints/agent_gen_5.zip

# 纯 CPU 训练
python scripts/train.py --device cpu

# 自定义环境数 / 步数
python scripts/train.py --envs 8 --steps 200000
```

### 人机对战

```bash
python webapp/server.py
```

浏览器打开 `http://127.0.0.1:5000`，即可与训练好的 AI 对战（中文 UI，主题"璀璨宝石"）。

### 运行测试

```bash
# 游戏规则单元测试
python -m pytest tests/test_rules.py -v

# 动作掩码单元测试
python -m pytest tests/test_action_mask.py -v

# 集成测试（1000 局随机游戏冒烟）
python tests/test_env.py
```

### 验证卡牌数据

```bash
python scripts/verify_cards.py
```

### 查看训练曲线

```bash
tensorboard --logdir logs
```

### 奖励函数架构

```
reward = γ · Φ(s') − Φ(s)          ← PBRS 骨架（望远镜求和至 −Φ(s₀)≈0）
       + build_bonus               ← 买到性价比 ≥ 0.5 的构筑牌 +0.20
       + block / restore           ← 卡对手 / 防被卡（恰好差 1 宝石的好牌）
       + terminal(±10 / 0)         ← 终局胜负
       + step / pass / hoarding    ← 小额罚项

势函数 Φ(s) = Strength(self) − Strength(opponent)

Strength(p) = w_point(n) · p.points + w_bonus(n) · bonus_alignment(p)

其中:
  · n = 代理已购牌数 — 冻结在单次 compute_reward 内
  · w_point: 0.05 → 0.15  (n 越大越重分数)
  · w_bonus: 0.15 → 0.03  (n 越大越轻引擎)
  · demand = 版面 T2/T3 中高性价比牌的加权颜色需求向量
  · bonus_alignment = demand · player.bonuses (点积)
```

## 核心参数

| 参数 | 值 | 说明 |
|------|----|------|
| n_steps | 1024 | 每环境每次 roll-out 步数 |
| batch_size | 256 | PPO 小批次大小 |
| n_epochs | 4 | 每次更新 PPO epoch 数 |
| lr | 5e-5 → 1e-5 | 余弦退火学习率 |
| γ | 0.99 | 折扣因子（PPO 与 PBRS 必须一致） |
| ent_coef | 0.08 → 0.01 | 熵系数线性退火 |
| 并行环境数 | 22 | SubprocVecEnv |
| 每代步数 | 500,000 | 环境交互步 |
| 总代数 | 100 | 约 50M 总步数 |
| 对手池大小 | 30 | ELO 加权采样 |
| max_turns | 200 | 单局最大回合（安全上限） |

## 技术亮点

- **势能奖励塑形 (PBRS)**：奖励 = γΦ(s') − Φ(s)，保证策略不变性，消除奖励塑形对最优策略的偏差。势函数权衡分数与引擎对齐度，并随购牌数动态切换重心。
- **持久并行环境**：Windows spawn 模式下 22 个 SubprocVecEnv 跨代复用，对手策略通过序列化 state_dict 热交换，避免每代 15–40s 的进程重启开销。
- **ELO 对手池**：维护最多 30 个历史检查点，混合采样策略（最新对手 50% + 均匀 10% + 随机动作 5% + ELO-softmax），确保训练对手多样性。
- **MaskablePPO**：51 个离散动作 + 布尔掩码，网络通过动作掩码自动忽略非法动作，加速收敛。
- **双座位交替训练**：代理在 P0/P1 间交替，确保回合标记特征的两个值都被充分看到。
