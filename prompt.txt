#plan
# 1. 我的目标
我想要训练一个基于强化学习（Reinforcement Learning）的摧残宝石（Splendor）智能体，采用路线 A：基于 Model-Free 的策略梯度算法（PPO），结合自对弈（Self-Play）进行迭代进化。

# 2. 核心上下文与数据
- 所有的卡牌数据已在 `cards_data.xlsx` 中给出。
- 每张卡牌包含：Level（等级1~3）、Bonus（产出宝石颜色）、Point（分值），以及购买所需的宝石成本（Cost，格式为包含各色数量的 JSON 字符串或独立列）。
- 这是一个双人、非完美/完美信息交织的、回合制零和博弈游戏。

# 3. 设计约束（严格遵守官方双人标准）
请在设计环境（Env）逻辑时，严格落实以下摧残宝石标准规则及边界：
1. **资源总数限制**：双人模式下，5种基础宝石（黑、白、红、蓝、绿）每种**只有 4 枚**；万能黄金（Gold）共 **5 枚**。
2. **拿取宝石限制**：
   - 玩家可选择拿取 3 个不同颜色的基础宝石。
   - 玩家可选择拿取 2 个相同颜色的基础宝石，但**只有当该颜色宝石在场上的存量 >= 4 时**才合法（因此双人局中只有满存量时才能执行此动作）。
3. **留牌（Reserve）与手牌限制**：
   - 玩家最多只能留 3 张卡牌在手里。手牌满 3 张后，禁止执行留牌动作。
   - 留牌时可获得 1 枚黄金。若场上黄金已拿光，玩家仍可留牌，但拿不到黄金。
4. **筹码持有上限**：玩家回合结束时，手里的**宝石总数（包含黄金）不能超过 10 枚**。如果超过，必须在该回合结束前选择多余的宝石退回环境。
5. **购牌减免机制（Bonus）**：玩家已购买的卡牌所提供的 Bonus，在后续购买新卡牌时直接充当永久折扣。
6. **胜负判定**：某位玩家的分数达到或超过 15 分时，触发最后一轮：如果是先手触发的，后手玩家还有最后一次完整行动必须完成；如果是后手玩家触发的，游戏直接结束（确保两人回合数相同）。该轮结束后，总分最高者获胜；若平分，则购买卡牌数量较少者获胜。
7. **不考虑任何贵族卡（Noble Cards）**。

# 4. 技术栈与架构设计
1. **环境接口**：使用 Gym/Gymnasium 接口标准定制开发 `SplendorEnv`。
2. **动作空间与掩码（Action Masking）**：
   - 游戏的动作空间高度动态且存在大量非法操作。必须设计并支持 **Action Masking** 机制。
   - 每一回合，环境需要根据当前状态（场面宝石、卡牌、玩家手牌和资产）计算出当前合法的动作掩码矩阵，将非法动作的概率强制设为 0，防止无效探索。
3. **状态空间（Observation Space）设计**：请为我设计一个紧凑且平坦化的多维向量或 Dict 空间，包含：场面公开卡牌、各堆剩余卡牌数、场面剩余宝石、双侧玩家的资产（各色筹码、各色Bonus、手牌、当前总分）。
4. **奖励塑造（Reward Shaping）**：为了缓解零和博弈稀疏奖励（Sparse Reward，仅在终局有 +1/-1）的问题，请设计合理的辅助奖励（例如：购买卡牌获得分数的即时正反馈、资产效率提升等），但要避免智能体陷入局部最优（如盲目刷筹码而不买牌）。
5. **训练框架**：优先考虑兼容现成的强化学习库（如 Ray/RLlib 或 Stable-Baselines3），以便于实现自对弈（Self-Play）训练管线。

# 5. 我的硬件配置（研发与训练环境优化说明）
请在编写数据加载、环境并行化（Vectorized Env）以及训练脚本时，充分利用我以下的硬件算力，避免代码写得过于单线程或发生显存溢出：
- **GPU (显卡)**: NVIDIA GeForce RTX 5070 Ti (16GB VRAM)
  - *优化建议*：支持较大的训练 Batch Size，神经网络模型（Policy/Value Net）可以适当加深或使用更宽的 MLP/Transformer 结构。支持高效的 Tensor 计算，请确保数据能顺畅地在 GPU 上流动。
- **CPU (处理器)**: 24核 24线程
  - *优化建议*：在强化学习中，环境采样通常是瓶颈。请在训练管线中**配置高并发的环境并行采样（Async Vector Env）**。建议设置多进程（Worker 数可设为 16~20），充分压榨 24 核 CPU 来高并发生成游戏对局样本，喂饱 GPU。
- **RAM (内存)**: 32GB
  - *优化建议*：多进程并发采样以及存储大型 Replay Buffer（经验回放池）时需要充足内存。请合理控制单个环境示例的内存占用，确保整体并发时不超过 32GB 阈值。
